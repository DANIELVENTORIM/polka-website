PolKA: Polynomial Key-based Architecture

Visão Geral

O PolKA é uma arquitetura de roteamento stateless baseada em:

  • Sistema de Numeração por Resíduos (RNS)
  • Teorema Chinês do Resto (TCR)
  • Aritmética Polinomial em Galois Fields (GF(2))

Princípios Técnicos

Componentes Chave

Elemento Descrição Formato
nodeID Identificador único do nó (polinômio irredutível) GF(2)[x]/⟨p(x)⟩
portID Identificador de porta (polinômio binário) bnxn + ... + b0
routeID Identificador de rota calculado via TCR CRT(nodeIDi, portIDi)

Fluxo de Operação

  1. Configuração: Controlador SDN atribui nodeIDs e portIDs
  2. Cálculo de Rota: Gera routeID usando TCR para caminho desejado
  3. Encapsulamento: Edge nodes adicionam routeID ao cabeçalho
  4. Forwarding: Nós core calculam portID = routeID mod nodeID
  5. Decapsulamento: Edge final remove routeID

Inovações Principais

Stateless Forwarding

Nós core não mantêm tabelas de roteamento - decisão via operação modular

Header Fixo

Cabeçalho de 128 bits independente do comprimento do caminho

Line Rate Processing

Implementação em hardware via CRC (Cyclic Redundancy Check)

MOD ≡ 2 SHIFTs + 1 CRC + 2 XORs
